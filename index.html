Cheat Sheet

Esta pagina esta dedicada a facilitar el acceso a los comandos para git y terminal en los sistemas operativos de linux, windows y mac

Comandos en Git

Estos comandos son una lista de los mas usados en Git y que te seran de mucha ayuda para tus proyectos:

Nombre Descripcion Escritura
Git configuration Establezca el nombre que se adjuntará a sus comits y tags. $ git config --global user.name “Your Name”
Git init Crear un nuevo repositorio local. Si se proporciona [nombre del proyecto], Git creará un nuevo directorio con el nombre de [nombre del proyecto] e inicializará un repositorio dentro de él. $ git init [project name]
Git clone Descarga un proyecto con todo el historial del repositorio remoto $ git clone [project url]
Git add Se usa para agregar archivos al área de preparación. git add [path]
Git commit Creará una instantánea de los cambios y la guardará en el directorio git. $ git commit "..."
Git status Muestra la lista de los archivos que se han cambiado junto con los archivos que están por ser preparados o confirmados. $ git status
Git push Se usa para enviar confirmaciones locales a la rama maestra del repositorio remoto. $ git push [--tags] [remote]
Git rebase Se usa para aplicar ciertos cambios de una rama en otra. $ git rebase [base]
Git fsck Realiza una comprobación de integridad del sistema de archivos git e identifica cualquier objeto corrupto $ git fsck [--tags]
Git prune Elimina los objetos que no tengan ningún apuntador entrante. $ git prune [-n] [-v] [--progress]
Git archive Le permite al usuario crear archivos zip o tar que contengan los constituyentes de un solo árbol de repositorio. $ git archive [--format= fmt] [--list]
Git gc Limpiará archivos innecesarios y optimizará el repositorio local. $ git gc [--aggressive] [--auto]
Git instaweb Te permite explorar tu repositorio local en la interfaz GitWeb. $ git instaweb [--local] [--httpd= httpd ]
Git K Muestra la interfaz gráfica para un repositorio local. $ gitk [ options ] [ revision range ] [--]
Git grep Le permite al usuario buscar frases y palabras específicas en los árboles de confirmación, el directorio de trabajo y en el área de preparación. $ git grep -n gmtime_r
Git cat-file Se usa para ver la información de tipo y tamaño de un objeto del repositorio. $ git cat-file type u object
Git ls-tree te permite ver un objeto de árbol junto con el nombre y modo de cada ítem, y el valor blob de SHA-1. $ git ls-tree [-d] [-r] [-t] [-l] [-z]
Git fetch Le permite al usuario buscar todos los objetos de un repositorio remoto que actualmente no se encuentran en el directorio de trabajo local. $ git fetch remote
Git show Se usa para mostrar información sobre cualquier objeto git. $ git show [options] [object]
Git stash Guardará momentáneamente los cambios que no están listos para ser confirmados. $ git stash
Git tag Marca commits específicos. Los desarrolladores lo usan para marcar puntos de lanzamiento como v1.0 y v2.0. $ git tag tagname
Git log Se usa para ver el historial del repositorio listando ciertos detalles de la confirmación. $ git log [options] [revision-range]
Git reset Sirve para resetear el index y el directorio de trabajo al último estado de confirmación. $ git reset --hard HEAD^
Git rm Se puede usar para remover archivos del index y del directorio de trabajo. $ git rm Documentation/
Git diff Se usa para hacer una lista de conflictos. $ git diff [options] [commit]
Git merge Se usa para fusionar una rama con otra rama activa: $ git merge new-feature
Git pull Fusiona todos los cambios que se han hecho en el repositorio remoto con el directorio de trabajo local. $ git pull --no-commit remote
Git branch Se usa para listar, crear o borrar ramas. $ git branch testing
Git remote Te permite ver todos los repositorios remotos. $ git remote [-v | --verbose]
Git checkout Crea ramas y te ayuda a navegar entre ellas. $ git checkout [-q] [-f] [-m] [branch]
